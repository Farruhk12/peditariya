# Техническое задание (ТЗ) — Образовательная платформа КДБ №1

**Проект:** peditariya-master  
**Назначение:** Образовательная веб-платформа для студентов педиатрии (Кафедра детских болезней №1, ТГМУ им. Абуали ибни Сино)  
**Дата ТЗ:** 16.03.2025

---

## 1. Общее описание

### 1.1 Тип проекта
Веб-приложение (SPA) — образовательная платформа для изучения педиатрии с интеграцией ИИ для ответов на вопросы, генерации карточек и оценки экзаменационных ответов.

### 1.2 Целевая аудитория
- **Студенты** 4 и 6 курсов факультетов Педиатрический и Лечебный
- **Администраторы** — преподаватели, добавляющие темы и экзаменационные вопросы

### 1.3 Текущий технологический стек

| Слой | Технологии |
|------|------------|
| Frontend | HTML5, CSS3, Vanilla JavaScript (ES6+), без фреймворков |
| Сервер | Node.js (встроенные `http`, `fs`, `path`) |
| ИИ | Google Gemini API (модель `gemini-2.0-flash`) |
| Хранение данных | Google Sheets + Google Apps Script |
| Шрифты | Google Fonts (IBM Plex Sans, Manrope) |

## 3. Функциональные требования

### 3.1 Для студентов

#### 3.1.1 Онбординг (первый вход)
- **Вход:** выбор курса (4 или 6) и факультета (Педиатрический / Лечебный)
- **Хранение:** `localStorage` под ключом `userPrefs`
- **Поведение:** после выбора загружаются темы, отфильтрованные по курсу и факультету
- **Возможность:** изменить выбор через кнопку «Изменить» в шапке

#### 3.1.2 Видеоматериалы
- **Вкладки:** «Общий» и «Патогенез» (если у темы есть обе ссылки)
- **Поддерживаемые источники:**
  - YouTube (watch, embed, shorts)
  - Vimeo
  - Google Drive (file/d/ID, open?id=ID)
  - Прямые ссылки на MP4, WebM, OGG
- **Рендер:** iframe для YouTube/Vimeo/Drive, `<video>` для прямых ссылок

#### 3.1.3 Описание тем
- **Формат:** Markdown
- **Поддержка:** заголовки (# ## ###), ссылки [текст](url), жирный **text**, курсив *text*
- **Конвертация:** функция `mdToHtml()` в app.js

#### 3.1.4 ИИ-чат
- **Назначение:** вопросы по теме, ответы через ИИ
- **API:** POST `/api/chat` на Node.js сервер
- **Промпт:** системный — «помощник для студентов педиатрии», без вступлений, 2–3 пункта
- **Контекст:** название темы + описание (до 8000 символов) + вопрос пользователя
- **Сохранение:** Q&A сохраняются в Google Sheets (лист «Вопросы_ИИ»)
- **Отображение:** история Q&A загружается с GAS (`action=qa`)

#### 3.1.5 Карточки (flashcards)
- **Генерация:** 12 карточек «вопрос–ответ» через ИИ
- **API:** POST `/api/chat` с `channel: 'cards'`, maxTokens: 2000
- **Формат ответа ИИ:** JSON `[{"q":"Вопрос?","a":"Ответ."}]`
- **Функции:** переворот карточки (клик/пробел), навигация (стрелки), прогресс-бар
- **Хранение:** `localStorage` — `cards_{topicId}` (карточки), `cards_o_{topicId}` (открытые)
- **Поздравление:** при просмотре всех карточек

#### 3.1.6 Экзаменационные вопросы
- **Отображение:** список вопросов по выбранной теме
- **Источник:** GAS `action=examQuestions`
- **Только просмотр:** без тестирования, просто список

#### 3.1.7 Проверить себя (self-test)
- **Логика:** до 4 случайных вопросов по текущей теме
- **Процесс:** по одному вопросу, текстовое поле для ответа, кнопки «Следующий» / «Пропустить»
- **Оценка:** POST `/api/grade` — ИИ оценивает каждый ответ по шкале 1–5 с комментарием
- **Результат:** средняя оценка, список оценок и комментариев по каждому вопросу

#### 3.1.8 Пройти экзамен
- **Логика:** 4 вопроса из разных тем курса (по одному на тему)
- **Источник:** GAS `action=examQuestionsAll` с фильтром course/faculty
- **Оценка:** аналогично «Проверить себя», через `/api/grade`
- **Контекст:** для каждого вопроса передаётся topicTitle и topicDesc

#### 3.1.9 Социальные функции
- **Лайки:** один лайк на тему с устройства (токен в localStorage)
- **Комментарии:** добавление комментария (имя + текст), отображение списка
- **Шаринг:** Web Share API или копирование ссылки `#topic={id}`

### 3.2 Для администраторов

#### 3.2.1 Авторизация
- **Пароль:** задаётся в Google Apps Script (Script Properties → `ADMIN_PASSWORD`)
- **Проверка:** при каждом POST-запросе addTopic, updateTopic, saveExamQuestions

#### 3.2.2 Добавление/редактирование тем
- **Поля:** название, описание (Markdown), ссылка видео общий, ссылка патогенез, курс, факультет
- **Действия:** addTopic (POST), updateTopic (POST) — с topicId для редактирования
- **Загрузка:** по ID темы подтягиваются данные для редактирования

#### 3.2.3 Экзаменационные вопросы
- **Ввод:** вставка текста из Word
- **Парсинг форматов:**
  - `@5Вопрос` — каждая строка с @5
  - `1. Вопрос` или `1) Вопрос` — нумерованный список
  - Fallback: параграфы по двойному переносу
- **Сохранение:** saveExamQuestions — заменяет все вопросы темы новым списком

## 4. API-спецификация

### 4.1 Google Apps Script (GAS)

**Базовый URL:** задаётся в `config.js` как `GAS_API`

#### GET-запросы (параметр `action`)

| action | Параметры | Ответ |
|--------|-----------|-------|
| topics | course, faculty | `{ topics: [...] }` |
| comments | topicId | `{ comments: [...] }` |
| reactions | topicId | `{ reactions: { likes: N } }` |
| qa | topicId | `{ qa: [...] }` |
| examQuestions | topicId | `{ questions: [...] }` |
| examQuestionsAll | course, faculty | `{ questions: [...] }` |

#### POST-запросы (JSON или form-urlencoded, поле `action`)

| action | Поля | Ответ |
|--------|------|-------|
| addComment | topicId, name, text | `{ ok: true }` |
| addQA | topicId, question, answer | `{ ok: true }` |
| like | topicId, token | `{ ok: true, likes: N }` или `{ already: true }` |
| addTopic | adminPassword, title, description, videoLink, videoLinkPathogen, course, faculty | `{ ok: true, topicId }` |
| updateTopic | adminPassword, topicId, title, ... | `{ ok: true }` |
| saveExamQuestions | adminPassword, topicId, topicTitle, questions (JSON array) | `{ ok: true, saved: N }` |

### 4.2 Node.js сервер (ИИ)

**Порт:** 3000 (или `process.env.PORT`)

#### POST /api/chat
- **Body:** `{ topicId, question, topicTitle, topicDesc, channel? }`
- **channel:** `'cards'` — для генерации карточек (больше токенов)
- **Ответ:** `{ answer: string }` или `{ error: string }`

#### POST /api/grade
- **Body:** `{ topicId?, topicTitle?, topicDesc?, studentId?, answers: [{ question, answer, topicTitle?, topicDesc? }] }`
- **Ответ:** `{ ok: true, grades: [{ grade, comment }], avgGrade: number }` или `{ ok: false, error }`

#### Статика
- GET `/*` — раздача HTML, CSS, JS, JSON, ico

## 6. ИИ-интеграция (для пересборки в другом ИИ)

### 6.1 Текущая реализация (Google Gemini)

- **Модель:** `gemini-2.0-flash`
- **Endpoint:** `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent`
- **Ключ:** `GEMINI_API_KEY` в `.env`

### 6.2 Промпты для переиспользования

#### Чат (ответ на вопрос студента)
```
Системный промпт:
Ты — помощник для студентов педиатрии ТГМУ. Отвечай на русском, медицински точно.
ВАЖНО: Давай сразу суть — без вступлений. Запрещено: «Привет!», «Давай разберёмся», «Итак», «Хороший вопрос» и т.п.
Формат: сразу по делу, 2–3 пунктами, без лишних слов.

Пользовательский промпт:
Тема: {topicTitle}
Контекст: {topicDesc}
---
Вопрос: {question}
```

#### Генерация карточек
```
Сгенерируй РОВНО 12 карточек «вопрос–ответ» по теме ниже.
Формат: чистый JSON-массив [{"q":"Вопрос?","a":"Ответ."}]. Без Markdown, без пояснений. Язык — русский.
Тема: {title}
Описание: {description}
```

#### Оценка ответов
```
Системный промпт:
Ты — строгий экзаменатор педиатрии. Оцени ответы как на реальном экзамене у будущих врачей. Будь максимально требовательным.

Критерии оценок:
- 5: Полный, точный ответ. Медицинская терминология, структура, все ключевые пункты. Без ошибок.
- 4: В целом верно, но есть неточности или незначительные пропуски.
- 3: Частичный ответ, пропущены важные аспекты, нет структуры.
- 2: Поверхностно, общие фразы, много ошибок или неверная трактовка.
- 1: Не ответил по существу, неверно, не по теме.

Не завывай оценки. Неполный ответ — максимум 3. «Думаю», «наверное», общие фразы без конкретики — 2 или ниже. Отсутствие ответа — 1.

Для каждого вопроса: оценка 1–5 и краткий, конкретный комментарий на русском (что упущено, что неверно).
Верни ТОЛЬКО JSON: [{"grade":N,"comment":"..."}]. Количество элементов = количеству вопросов.

Пользовательский промпт:
{context} + список вопросов и ответов студента
```

### 6.3 Требования к замене ИИ

При пересборке на другой ИИ (OpenAI, Claude, локальная модель и т.д.) необходимо реализовать:

1. **Функция `callAI(systemPrompt, userPrompt, maxTokens, temp)`** — возвращает строку ответа
2. **Эндпоинты:**
   - `/api/chat` — принять `{ topicId, question, topicTitle, topicDesc, channel? }`, вернуть `{ answer }`
   - `/api/grade` — принять `{ answers, topicTitle?, topicDesc? }`, вернуть `{ ok, grades, avgGrade }`
3. **Формат grades:** массив `[{ grade: 1-5, comment: string }]`
4. **Парсинг JSON** в ответах (карточки, оценки) — извлечение `[...]` через regex при необходимости

## 10. Рекомендации для пересборки

### 10.1 Замена ИИ
- Сохранить структуру промптов и форматы ответов
- Реализовать `callAI` под выбранный API (OpenAI, Anthropic, локальный Ollama и т.д.)
- Адаптировать парсинг JSON под возможные отличия в выводе модели

### 10.2 Замена бэкенда данных
- Google Sheets можно заменить на PostgreSQL, MongoDB, Supabase и т.д.
- Сохранить структуру API (action-based или REST) для совместимости с фронтендом
- Миграция: экспорт из Sheets в CSV → импорт в новую БД

### 10.3 Модернизация фронтенда
- Можно переписать на React/Vue/Svelte
- Сохранить логику: онбординг → темы → вкладки (видео, карточки, экзамен и т.д.)
- API-вызовы остаются теми же при сохранении контрактов

### 10.4 Деплой
- Текущий: локальный Node.js + GAS в облаке
- Варианты: Vercel/Netlify (статический фронт) + отдельный сервер ИИ; или монолит на Railway/Render
